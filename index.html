Introduction to MySQL

What is MySQL?

MySQL is an open-source relational database management system (RDBMS) that uses SQL (Structured Query Language) to manage and manipulate data.

  • SQL keywords are NOT case-sensitive
  • Used by many popular websites and applications
  • Supports large databases with high performance
  • Provides data security and integrity
Key Features
  • Relational Database: Stores data in tables with relationships
  • ACID Compliance: Ensures data reliability
  • Cross-Platform: Works on Windows, Linux, macOS
  • Scalable: Handles small to large applications
  • Secure: User authentication and access control
Tip: SQL keywords are not case-sensitive: SELECT = select = Select

Database Operations

Show Databases

Display all databases in the MySQL server:

SHOW DATABASES;
Create Database

Create a new database:

CREATE DATABASE myDb;
Use Database

Select a database to work with:

USE myDb;
Drop Database

Delete a database:

DROP DATABASE myDb;
Warning: This permanently deletes the database and all its data!
Read-Only Mode

Set database to read-only mode:

-- Set to read-only
ALTER DATABASE mydb READ ONLY = 1;

-- Set back to read-write
ALTER DATABASE mydb READ ONLY = 0;

Table Management

Create Table

Create a new table with columns:

CREATE TABLE employees(
    employee_id INT,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    hourly_pay DECIMAL(5,2),
    hire_date DATE
);
View Table Structure
SELECT * FROM employees;
Rename Table
RENAME TABLE employees TO workers;
Drop Table
DROP TABLE employees;
Alter Table

Add a new column:

ALTER TABLE employees
ADD phone_number VARCHAR(15);

Rename a column:

ALTER TABLE employees
RENAME COLUMN phone_number TO email;

Modify column data type:

ALTER TABLE employees
MODIFY COLUMN email VARCHAR(100);

Change column position:

-- Move column after last_name
ALTER TABLE employees
MODIFY email VARCHAR(100)
AFTER last_name;

-- Move column to first position
ALTER TABLE employees
MODIFY email VARCHAR(100)
FIRST;

Drop a column:

ALTER TABLE employees
DROP COLUMN email;

Data Manipulation Language (DML)

Insert Data

Insert a single row:

INSERT INTO employees
VALUES (1, "Okeyo", "Oyoo", "okoyo@gmail.com", 30.05, "2025-01-02");

Insert multiple rows:

INSERT INTO employees
VALUES 
    (2, "Lil", "Durk", "lil@gmail.com", 40.5, "2025-01-23"),
    (3, "Lil", "Baby", "lil@gmail.com", 40.5, "2025-01-23"),
    (4, "king", "von", "king@gmail.com", 40.5, "2025-01-23"),
    (5, "Lil", "wayne", "lil@gmail.com", 40.5, "2025-01-23");

Insert specific columns only:

INSERT INTO employees (employee_id, first_name, last_name)
VALUES (6, "Gta", "Love");
Select Data

Select all columns:

SELECT * FROM employees;

Select specific columns:

SELECT last_name, first_name
FROM employees;

Using WHERE clause:

-- Equality check
SELECT * FROM employees 
WHERE employee_id = 1;

-- Comparison operators
SELECT * FROM employees 
WHERE hourly_pay >= 15;

-- NULL checks
SELECT * FROM employees
WHERE hourly_pay IS NULL;

SELECT * FROM employees
WHERE hourly_pay IS NOT NULL;

Logical operators:

SELECT * FROM employees
WHERE hourly_pay = 30.5 OR job = "Manager";

-- Other operators: !=, <, >, <=, >=, NOT, BETWEEN... AND, IN(...)

Wildcards:

-- % : Matches any sequence of characters
SELECT * FROM employees
WHERE first_name LIKE "d%";  -- Starts with 'd'

SELECT * FROM employees
WHERE last_name LIKE "%s";   -- Ends with 's'

-- _ : Matches a single character
SELECT * FROM employees
WHERE job LIKE "_ook";

-- Combined
SELECT * FROM employees
WHERE job LIKE "_a%";

Sorting with ORDER BY:

-- Ascending (default)
SELECT * FROM employees
ORDER BY employee_id;

-- Descending
SELECT * FROM employees
ORDER BY employee_id DESC;

Limiting results:

-- Get first row
SELECT * FROM employees 
LIMIT 1;

-- Get top 2 after sorting
SELECT * FROM employees 
ORDER BY last_name DESC
LIMIT 2;
Update Data

Update single column:

UPDATE employees
SET hire_date = "2025-09-24"
WHERE first_name = "Gta";

Update multiple columns:

UPDATE employees
SET hire_date = "2025-09-24",
    hourly_pay = 34.50
WHERE first_name = "Gta" AND last_name = "Love";
Important: Always use WHERE clause with UPDATE or you'll update ALL rows! 
-- Disable safe mode for updates without WHERE
SET SQL_SAFE_UPDATES = 0;

-- Enable safe mode (recommended)
SET SQL_SAFE_UPDATES = 1;
Delete Data
DELETE FROM employees
WHERE employee_id = 2;
Warning: Without WHERE clause, ALL rows will be deleted!
Transaction Control
-- Turn off auto-commit
SET AUTOCOMMIT = OFF;

-- Rollback changes if you forget WHERE clause
ROLLBACK;

-- Commit changes to save permanently
COMMIT;
Working with Dates
INSERT INTO test
VALUES(
    (CURRENT_DATE()),
    (CURRENT_TIME()),
    (NOW())
);

Constraints

UNIQUE Constraint

Create table with UNIQUE constraint:

CREATE TABLE products(
    product_id INT,
    product_name VARCHAR(50) UNIQUE,
    price DECIMAL(5,2)
);

Add UNIQUE constraint to existing table:

ALTER TABLE products
ADD CONSTRAINT UNIQUE(product_name);
NOT NULL Constraint

Ensures a column cannot have NULL values:

CREATE TABLE customers(
    customer_id INT NOT NULL,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100)
);
CHECK Constraint

Create table with CHECK constraint:

CREATE TABLE employees(
    employee_id INT,
    hourly_pay DECIMAL(5,2),
    CONSTRAINT chk_hourly_pay CHECK (hourly_pay >= 10.00)
);

Add CHECK constraint to existing table:

ALTER TABLE employees
ADD CONSTRAINT chk_hourly_pay CHECK(hourly_pay >= 10.00);

Drop CHECK constraint:

ALTER TABLE employees
DROP CHECK chk_hourly_pay;
DEFAULT Constraint

Create table with DEFAULT value:

CREATE TABLE products(
    product_id INT,
    price DECIMAL(4,2) DEFAULT 0.00
);

Add DEFAULT to existing column:

ALTER TABLE products
ALTER price SET DEFAULT 0.00;

Insert with DEFAULT values:

INSERT INTO products(product_id, product_name)
VALUES (5, "napkins"),
       (6, "straws"),
       (7, "razor");
PRIMARY KEY Constraint

Create table with PRIMARY KEY:

CREATE TABLE transactions(
    transaction_id INT PRIMARY KEY,
    amount DECIMAL(5,2)
);

Add PRIMARY KEY to existing table:

ALTER TABLE transactions
ADD CONSTRAINT PRIMARY KEY (transaction_id);
PRIMARY KEY columns are automatically NOT NULL and UNIQUE.
AUTO_INCREMENT
CREATE TABLE transactions(
    transaction_id INT PRIMARY KEY AUTO_INCREMENT,
    amount DECIMAL(5,2)
);

Set AUTO_INCREMENT starting value:

ALTER TABLE transactions
AUTO_INCREMENT = 1000;
FOREIGN KEY Constraint

Create table with FOREIGN KEY:

CREATE TABLE transactions(
    transaction_id INT PRIMARY KEY AUTO_INCREMENT,
    amount DECIMAL(5,2),
    customer_id INT,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

Add FOREIGN KEY to existing table:

ALTER TABLE transactions
ADD CONSTRAINT fk_customer_id
FOREIGN KEY (customer_id) REFERENCES customers(customer_id);

Drop FOREIGN KEY:

ALTER TABLE transactions
DROP FOREIGN KEY transactions_ibfk_1;

Foreign key checks:

-- Disable foreign key checks
SET foreign_key_checks = 0;

-- Enable foreign key checks
SET foreign_key_checks = 1;

ON DELETE options:

-- ON DELETE SET NULL: Replace foreign key with NULL when referenced row is deleted
ALTER TABLE transactions
ADD CONSTRAINT fk_customer_id
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
ON DELETE SET NULL;

-- ON DELETE CASCADE: Delete row when referenced row is deleted
ALTER TABLE transactions
ADD CONSTRAINT fk_customer_id
FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
ON DELETE CASCADE;

Joins & Functions

JOIN Types

INNER JOIN (returns matching rows):

SELECT * 
FROM transactions 
INNER JOIN customers
ON transactions.customer_id = customers.customer_id;

Select specific columns with INNER JOIN:

SELECT transaction_id, last_name, first_name, amount
FROM transactions 
INNER JOIN customers
ON transactions.customer_id = customers.customer_id;

LEFT JOIN (all rows from left table):

SELECT * 
FROM transactions 
LEFT JOIN customers
ON transactions.customer_id = customers.customer_id;

RIGHT JOIN (all rows from right table):

SELECT * 
FROM transactions 
RIGHT JOIN customers
ON transactions.customer_id = customers.customer_id;
SQL Functions

Aggregate functions:

-- Count rows
SELECT COUNT(amount) FROM transactions;

-- Maximum value
SELECT MAX(amount) AS maximum FROM transactions;

-- Minimum value
SELECT MIN(amount) AS minimum FROM transactions;

-- Average value
SELECT AVG(amount) AS average FROM transactions;

-- Sum of values
SELECT SUM(amount) AS total FROM transactions;

String functions:

-- Concatenate strings
SELECT CONCAT(last_name, " ", first_name) AS full_name
FROM customers;
UNION Operator
-- Combine results from multiple SELECT statements
SELECT first_name, last_name FROM customers
UNION
SELECT first_name, last_name FROM employees;
UNION requires the same number of columns in all SELECT statements.
Self Join
-- Joining a table with itself
SELECT a.customer_id, a.first_name, a.last_name,
       CONCAT(b.first_name, " ", b.last_name) AS "referred_by"
FROM customers AS a
INNER JOIN customers AS b
ON a.referral_id = b.customer_id;

Employee supervision example:

SELECT a.employee_id, a.first_name, a.last_name,
       CONCAT(b.last_name, " ", b.first_name) AS "supervised_by"
FROM employees AS a
INNER JOIN employees AS b
ON a.supervisor_id = b.employee_id
ORDER BY employee_id;

Advanced SQL Features

Views

Create a view (virtual table):

CREATE VIEW employees_attendance AS
SELECT first_name, last_name
FROM employees;

Drop a view:

DROP VIEW employees_attendance;
Indexes

Show all indexes on a table:

SHOW INDEXES FROM customers;

Create an index:

CREATE INDEX last_name_idx
ON customers(last_name);

Create multi-column index:

CREATE INDEX last_name_first_name_idx
ON customers(last_name, first_name);

Drop an index:

ALTER TABLE customers
DROP INDEX last_name_idx;
Subqueries

Subquery in SELECT:

SELECT last_name, first_name, hourly_pay, 
       (SELECT AVG(hourly_pay) FROM employees) AS average
FROM employees;

Subquery in WHERE clause:

SELECT customer_id, first_name, last_name 
FROM customers
WHERE customer_id IN
      (SELECT DISTINCT customer_id 
       FROM transactions 
       WHERE customer_id IS NOT NULL);
GROUP BY

Group rows by a column:

SELECT SUM(amount) AS total_amount, customer_id
FROM transactions
GROUP BY customer_id;

Using HAVING clause (filter groups):

SELECT COUNT(amount), customer_id
FROM transactions
GROUP BY customer_id
HAVING COUNT(amount) > 1 AND customer_id IS NOT NULL;

WITH ROLLUP (adds summary rows):

SELECT COUNT(amount), customer_id
FROM transactions
GROUP BY customer_id WITH ROLLUP;
Stored Procedures

Create a stored procedure:

DELIMITER $$

CREATE PROCEDURE get_customers()
BEGIN 
    SELECT * FROM customers;
END $$

DELIMITER ;

Call a stored procedure:

CALL get_customers();

Drop a stored procedure:

DROP PROCEDURE get_customers;

Procedure with parameters:

DELIMITER $$

CREATE PROCEDURE find_customer(IN id INT)
BEGIN
    SELECT customer_id, last_name, first_name 
    FROM customers
    WHERE customer_id = id;
END $$

DELIMITER ;

-- Call with parameter
CALL find_customer(3);
Triggers

Create a trigger (BEFORE UPDATE):

CREATE TRIGGER before_hourly_pay_update
BEFORE UPDATE ON employees
FOR EACH ROW 
SET NEW.salary = (NEW.hourly_pay * 2080);

Create a trigger (BEFORE INSERT):

CREATE TRIGGER before_hourly_pay_insert
BEFORE INSERT ON employees
FOR EACH ROW
SET NEW.salary = (NEW.hourly_pay * 2080);

Show all triggers:

SHOW TRIGGERS;
Triggers automatically execute when an event (INSERT, UPDATE, DELETE) occurs.

Database Connection with Node.js

Prerequisites: Ensure you have Node.js installed and initialize your project with npm init before proceeding.
MySQL Connection
mysql2 package
Installation
npm install mysql2 dotenv
Connection Setup (db.js)
import mysql from "mysql2/promise";
import dotenv from "dotenv";

dotenv.config();

// Create a connection pool for better performance
export const db = mysql.createPool({
    host: process.env.DB_HOST || "localhost",
    user: process.env.DB_USER || "root",
    password: process.env.DB_PASSWORD || "",
    database: process.env.DB_NAME || "mydatabase",
    waitForConnections: true,
    connectionLimit: 10,
    queueLimit: 0
});

// Test connection function
export async function testConnection() {
    try {
        const connection = await db.getConnection();
        console.log(" MySQL database connected successfully!");
        connection.release();
        return true;
    } catch (error) {
        console.error(" MySQL connection error:", error.message);
        return false;
    }
}

DB_PASSWORD=yourpassword
DB_NAME=yourdatabase
Usage Examples

SELECT Query:

async function getUsers() {
    try {
        const [rows] = await db.query("SELECT * FROM users WHERE active = ?", [1]);
        return rows;
    } catch (error) {
        console.error("Error fetching users:", error);
        throw error;
    }
}

INSERT Query:

async function createUser(name, email) {
    try {
        const [result] = await db.query(
            "INSERT INTO users (name, email, created_at) VALUES (?, ?, NOW())",
            [name, email]
        );
        return result.insertId;
    } catch (error) {
        console.error("Error creating user:", error);
        throw error;
    }
}

UPDATE Query:

async function updateUser(id, data) {
    try {
        const [result] = await db.query(
            "UPDATE users SET name = ?, email = ? WHERE id = ?",
            [data.name, data.email, id]
        );
        return result.affectedRows;
    } catch (error) {
        console.error("Error updating user:", error);
        throw error;
    }
}

Transaction Example:

async function transferFunds(fromId, toId, amount) {
    const connection = await db.getConnection();
    
    try {
        await connection.beginTransaction();
        
        // Deduct from sender
        await connection.query(
            "UPDATE accounts SET balance = balance - ? WHERE id = ?",
            [amount, fromId]
        );
        
        // Add to receiver
        await connection.query(
            "UPDATE accounts SET balance = balance + ? WHERE id = ?",
            [amount, toId]
        );
        
        await connection.commit();
        console.log("Transaction completed successfully");
    } catch (error) {
        await connection.rollback();
        console.error("Transaction failed:", error);
        throw error;
    } finally {
        connection.release();
    }
}
PostgreSQL Connection
pg package
Installation
npm install pg dotenv
Connection Setup (db.js)
import { Pool } from "pg";
import dotenv from "dotenv";

dotenv.config();

// PostgreSQL connection pool
export const db = new Pool({
  connectionString: process.env.DATABASE_URL,
  // For secure connections (like Supabase, AWS RDS, etc.)
  ssl: process.env.NODE_ENV === 'production' ? {
    rejectUnauthorized: false
  } : false,
  max: 20, // maximum number of clients in the pool
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

// Test connection function
export async function testPostgresConnection() {
    try {
        const client = await db.connect();
        console.log(" PostgreSQL database connected successfully!");
        client.release();
        return true;
    } catch (error) {
        console.error(" PostgreSQL connection error:", error.message);
        return false;
    }
}

Usage Examples

SELECT Query:

async function getUsers() {
    try {
        const result = await db.query(
            "SELECT * FROM users WHERE active = $1",

        );
        return result.rows;

    }
}

INSERT Query:

async function createUser(name, email) {
    try {
        const result = await db.query(
            "INSERT INTO users (name, email, created_at) VALUES ($1, $2, NOW()) RETURNING id",
            [name, email]
        );
        return result.rows[0].id;
    } catch (error) {
        console.error("Error creating user:", error);
        throw error;
    }
}

UPDATE Query:

async function updateUser(id, data) {
    try {
        const result = await db.query(
            "UPDATE users SET name = $1, email = $2 WHERE id = $3 RETURNING *",

        );
        return result.rows[0];
    } catch (error) {
        console.error("Error updating user:", error);
        throw error;
    }
}

Transaction Example:

async function transferFunds(fromId, toId, amount) {
    const client = await db.connect();
    
    try {
        await client.query('BEGIN');
        
        // Deduct from sender
        await client.query(
            "UPDATE accounts SET balance = balance - $1 WHERE id = $2",
            [amount, fromId]
        );
        
        // Add to receiver
        await client.query(
            "UPDATE accounts SET balance = balance + $1 WHERE id = $2",
            [amount, toId]
        );
        
        await client.query('COMMIT');
        console.log("Transaction completed successfully");
    } catch (error) {
        await client.query('ROLLBACK');
        console.error("Transaction failed:", error);
        throw error;
    } finally {
        client.release();
    }
}
MySQL vs PostgreSQL Comparison
Feature MySQL PostgreSQL
Parameter Placeholder ? (positional) $1, $2, $3 (numbered)
RETURNING Clause LAST_INSERT_ID() or multiple queries needed RETURNING * supported in INSERT/UPDATE
Connection Method mysql.createPool() new Pool()
Result Structure [rows, fields] array destructuring { rows, rowCount } object
Transaction Syntax BEGIN; ... COMMIT; or pool methods BEGIN; ... COMMIT;
Example Query INSERT ... VALUES (?, ?) INSERT ... VALUES ($1, $2) RETURNING id
Best Practices
  • Always use parameterized queries to prevent SQL injection
  • Use connection pools for production applications
  • Handle connection errors and implement retry logic
  • Close connections properly in finally blocks
  • Use environment variables for sensitive data (never hardcode credentials)
